cs-Solidarity · Project Video04 / FOUNDATION
THREE PRINCIPLES

先抓住三个词：
线程安全、反检测、远程管理。

后面所有功能，其实都围绕这三个设计点展开。它们不是配角，而是这个项目能长期跑下去的前提。

01线程安全发送请求先入队，主线程统一操作微信 GUI。
02反检测鼠标、键盘和窗口行为加入真人化扰动。
03远程管理Agent 主动连出，公网 Web 面板远程维护。
cs-Solidarity · Project Video05 / CAPABILITIES
WHAT IT DOES

能力不是堆出来的，
而是四条稳定的输出链路。

游戏状态监控、智能对话与 Agent、信息与预警推送、Web 远程管理，都是“采集 → 处理 → 回到微信”的同一种结构。

01 / STEAM游戏状态监控好友在线追踪 · CS2 新闻 · 完美世界战绩 · 每日报告
02 / AI智能对话与 Agent命令式任务入口 + 自然群聊人格回复
03 / PUSH信息与预警推送财经、新闻、地震、气象、海啸等多源信息
04 / OPS远程管理面板状态、日志、配置、文件、实例和控制能力
cs-Solidarity · Project Video06 / SYSTEM OVERVIEW
WHO TALKS TO WHOM

这些功能看起来很散，
但最后共用一个微信自动化核心。

外部数据源、业务实例、Bot API 和 Web 管理请求都不是各自为战，最终真正操作微信的动作，仍然统一回到主线程。

cs-Solidarity · Project Video07 / ARCHITECTURE
CORE / INSTANCES / EXTERNAL

它不是把功能硬塞进主循环，
而是拆成三层去接。

Core 层负责收发和调度，Instances 层负责业务，External 层负责外部接入。新功能只要写成 BaseInstance 子类就能挂进来。

cs-Solidarity · Project Video08 / INSTANCES
SIX INSTANCE MODULES

功能不同，
接入方式却故意做成一致。

SteamAuto、ChatAuto、KouriChat、InfoPush、DailyAuto、DisasterWarning 都只负责业务处理；真正发送时，要么自己入队，要么经 Agent 调用 Bot API 入队。

01SteamAuto状态、新闻、战绩、日报
02ChatAuto命令触发 Agent 任务
03KouriChat人设、记忆、上下文回复
04InfoPush新闻、金价、股票
05DailyAuto固定时间提醒
06DisasterWarning灾害预警定向推送
cs-Solidarity · Project Video09 / STEAMAUTO
STATUS DIFF TO GROUP PUSH

SteamAuto 的核心不是抓数据，
而是把状态差异翻译成群聊消息。

开始游戏、离开游戏、完美世界战绩、每日统计图，这些都不是手工写死的文案，而是由状态 diff 和模板拼出来的结果。

cs-Solidarity · Project Video10 / ANTI-DETECT
HUMAN-LIKE GUI AUTOMATION

真正像人一样操作微信的细节，
不应该散在每个功能里。

鼠标路径、点击等待、键盘节奏、窗口切换和最小化策略统一放进 Core 层。实例只关心业务，不关心怎么点微信。

cs-Solidarity · Project Video11 / MAIN LOOP
FIVE STEPS

主循环不是不停轮询，
而是有明确优先级的五步流程。

它先检查维护窗口，再处理 Web 消息，再消费发送队列，最后才去读新消息和分发，用顺序降低聊天窗口切换带来的竞态。

01检查维护窗口
02处理 Web 消息
03消费发送队列
04接收和分发新消息
05空闲时最小化微信窗口
cs-Solidarity · Project Video12 / ROUTING
EXCLUSIVE + BROADCAST

命令式功能和自然聊天能共存，
靠的是消息路由，不是侥幸。

前缀命令交给独占实例，普通聊天广播给上下文监听模块。这样 ChatAuto 和 KouriChat 不会互相抢消息。

cs-Solidarity · Project Video13 / REMOTE OPS
AGENT CONNECTS OUT

内网 Windows Bot 不需要开公网端口，
也能被远程维护。

Agent 主动连接公网 Server，浏览器只访问 Server，再由 Server 用 WebSocket 把操作转回 Bot。这就是整套远程运维的内核。

cs-Solidarity · Project Video14 / WEB PANEL
OBSERVABILITY MATTERS

Web 面板不是旁路工具，
而是把运维能力真正接进 Bot。

Dashboard、Config、Control、Logs、Files、Chat 不只是“能看”，而是会实际走到 Agent 转发、发送队列和微信窗口这一整条链路上。

Dashboard在线状态
Logs实时日志
Control启动停止
Chat同步微信
cs-Solidarity · Project Video15 / FINAL TAKEAWAY
SUMMARY

它不是一个自动发消息的小脚本，
而是一套微信自动化框架。

线程安全、模块化扩展、远程可运维，是这个项目真正值得被保留下来的工程内核。

01 / THREAD线程安全后台实例只入队，主线程统一操作微信 GUI。
02 / EXTEND模块化扩展Steam、AI、推送、预警都能作为独立实例接入。
03 / OPS远程可运维Agent 主动连接 Server，内网 Bot 也能公网管理。
