# cs-Solidarity — Steam status → WeChat notifier

简要：该项目监控 Steam 好友游戏状态，并将事件推送到指定的微信聊天（例如“文件传输助手”或群组）。项目基于本地 `pywechat` 与 `wxauto` 自动化库，适用于 Windows 环境。

快速开始
- 安装依赖：
```
pip install -r requirements.txt
```
- 配置：编辑 `config.json`（示例见仓库）或为每个实例准备单独的配置文件（`account/*.json`）。
- 运行：
```
python main.py
```

运行说明
- 默认模式（推荐）：主进程负责所有微信 GUI 自动化操作；各个 `SteamAuto` 实例在后台线程检测状态并把要发送的消息放入队列，主线程从队列读取并调用发送函数，避免多线程直接操控 UI 导致失败。
- 若需并行发送、隔离或在多台机器运行多个实例，请考虑多进程或远程 IPC（详见下节）。

配置要点
- `config.json` 包含 `instances`、`check_interval`、`steam_api_key`、`steam_id`、`wechat_groups` 等字段。
- `account/` 下存放示例实例配置文件。

常见问题
- 为什么不能在多线程中直接发消息？
    - 微信消息发送依赖 GUI 自动化（`pyautogui` / `pywinauto`），这些操作不是线程安全，必须在单一主线程或专用发送进程中执行。

并发运行多个实例（建议）
- 方案 A（当前实现）：线程 + 队列 — 各实例线程检测并入队，主线程发送。优点：实现简单；缺点：发送为串行。已经内置于 `main.py`。
- 方案 B（推荐用于并发发送）：多进程 + 发送进程 — 每个实例在独立进程检测或运行，所有发送请求经 IPC 转发到专用发送进程，由其执行 GUI 操作，避免线程/进程间的 UI 冲突。

开发/调试
- 在修改自动化逻辑时，请确保微信客户端窗口能被脚本访问，并避免在运行时手动干预鼠标/键盘。
