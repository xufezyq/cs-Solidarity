# KoriChat 指令完整说明

## 📋 指令列表

KoriChat 支持多种指令，分为以下几类：

### 一、调试命令（以 `/` 开头）

这些命令用于调试和管理机器人的记忆、状态等。

| 指令 | 功能说明 | 使用示例 | 备注 |
|------|---------|---------|------|
| `/help` | 显示帮助信息 | `/help` | 查看所有可用命令 |
| `/mem` | 显示当前角色的记忆 | `/mem` | 查看核心记忆和短期记忆（最近 5 轮对话） |
| `/reset` | 重置当前角色的最近记忆 | `/reset` | 清空短期记忆文件 |
| `/clear` | 清空当前角色的核心记忆 | `/clear` | 清空核心记忆文件 |
| `/context` | 清空当前角色的对话上下文 | `/context` | 清空 LLM 对话历史 |
| `/gen_core_mem` | 手动生成核心记忆 | `/gen_core_mem` | 基于对话上下文更新核心记忆 |
| `/exit` | 退出调试模式 | `/exit` | 退出调试状态 |

### 二、内容生成命令

这些命令会基于角色人设生成特定类型的内容。

| 指令 | 功能说明 | 使用示例 | 输出内容 |
|------|---------|---------|---------|
| `/diary` | 生成角色小日记 | `/diary` | 角色的一天记录、心情日记 |
| `/state` | 查看角色状态 | `/state` | 角色的当前状态、情绪、想法 |
| `/letter` | 角色给你写的信 | `/letter` | 角色写给用户的信件内容 |
| `/list` | 角色的备忘录 | `/list` | 角色的待办事项、备忘录 |
| `/pyq` | 角色的朋友圈 | `/pyq` | 角色会发的朋友圈内容 |
| `/gift` | 角色想送的礼物 | `/gift` | 角色想送给用户的礼物清单 |
| `/shopping` | 角色的购物清单 | `/shopping` | 角色想买的物品清单 |

## 🎯 指令详细说明

### 1. 调试命令详解

#### `/help` - 显示帮助信息

**功能**：显示所有可用的调试命令列表

**返回示例**：
```
调试模式命令:
- /help: 显示此帮助信息
- /mem: 显示当前角色的记忆
- /reset: 重置当前角色的最近记忆
- /clear: 清空当前角色的核心记忆
- /context: 清空当前角色的对话上下文
- /diary: 生成角色小日记
- /state: 查看角色状态
- /letter: 角色给你写的信
- /list: 角色的备忘录
- /pyq: 角色的朋友圈
- /gift: 角色想送的礼物
- /shopping: 角色的购物清单
- /exit: 退出调试模式
```

#### `/mem` - 显示当前角色的记忆

**功能**：查看角色的核心记忆和短期记忆

**返回示例**：
```
核心记忆:
用户是主人，对我很好。我们每天都會聊天，分享彼此的生活。
主人喜欢玩游戏，尤其是 CS2。我会关心主人的身体状况，提醒他休息。

短期记忆:
用户：今天好累啊
回复：主人，您辛苦了！快休息一下吧，我给您按摩按摩~ (✪ω✪)

用户：明天要上班
回复：呜呜，主人要上班呢，那要注意休息哦~ 下班后我来陪您聊天！
```

**用途**：
- 了解角色记住了什么
- 检查记忆是否正常
- 调试记忆系统

#### `/reset` - 重置当前角色的最近记忆

**功能**：清空角色的短期记忆文件

**返回示例**：
```
已重置 ATRI 的最近记忆
```

**用途**：
- 清除最近的对话记录
- 重新开始对话
- 修复记忆错误

**注意**：只清空短期记忆，核心记忆不会受影响

#### `/clear` - 清空当前角色的核心记忆

**功能**：清空角色的核心记忆文件

**返回示例**：
```
已清空 ATRI 的核心记忆
```

**用途**：
- 清除角色的核心记忆
- 重新建立角色认知
- 修复记忆错误

**注意**：核心记忆是角色最重要的记忆，清空后角色会"忘记"重要信息

#### `/context` - 清空当前角色的对话上下文

**功能**：清空 LLM 的对话历史

**返回示例**：
```
已清空对话上下文
```

**用途**：
- 清除 LLM 的上下文记忆
- 开始全新的对话
- 解决上下文过长导致的错误

**注意**：这只清空 LLM 的临时上下文，不影响记忆文件

#### `/gen_core_mem` - 手动生成核心记忆

**功能**：基于当前对话上下文，手动生成/更新核心记忆

**返回示例**：
```
成功更新核心记忆
```

**用途**：
- 强制更新核心记忆
- 基于最新对话生成记忆
- 优化记忆内容

#### `/exit` - 退出调试模式

**功能**：退出调试模式

**返回示例**：
```
已退出调试模式
```

## 🔧 指令使用技巧

### 1. 命令格式

- 所有命令都以 `/` 开头
- 命令不区分大小写（`/HELP` 和 `/help` 效果相同）
- 命令后面可以跟参数（部分命令支持）

### 2. 使用场景

#### 私聊中使用
直接在私聊中发送命令即可：
```
/mem
/reset
/diary
```

#### 群聊中使用
在群聊中，命令会被识别，但回复可能会@发送者：
```
@机器人 /mem
@机器人 /state
```

### 3. 异步生成

部分内容生成命令（如 `/diary`、`/state` 等）支持异步生成：
- 发送命令后，机器人会立即回复"正在生成..."
- 生成完成后，会再次发送消息包含生成内容
- 这样不会阻塞其他消息处理

### 4. 命令组合

可以将多个命令组合使用：
```
# 先查看记忆，再重置
/mem
/reset

# 先查看状态，再清空上下文
/state
/context
```

## ⚠️ 注意事项

### 1. 权限控制

- 所有命令都可以被任何用户调用
- 建议只在私聊中使用调试命令
- 群聊中建议限制部分命令的使用

### 2. 数据安全

- `/reset`、`/clear` 等命令会删除数据，请谨慎使用
- 执行删除操作前，建议先备份记忆
- 核心记忆清空后无法恢复

### 3. 性能考虑

- 内容生成命令会调用 LLM API，消耗 token
- 频繁调用可能导致 API 限流
- 建议在需要时才使用生成命令

### 4. 命令冲突

- 如果用户消息以 `/` 开头，会被识别为命令
- 如果确实需要发送以 `/` 开头的普通消息，可以在前面加空格
- 例如：` /test`（前面有空格）会被当作普通消息

---

## 📊 命令响应时间

| 命令类型 | 响应时间 | 说明 |
|---------|---------|------|
| 调试命令（`/mem`、`/reset` 等） | 即时（<1 秒） | 直接操作文件 |
| 内容生成命令（`/diary`、`/state` 等） | 较慢（3-10 秒） | 需要调用 LLM API |
| 异步内容生成 | 后台处理 | 立即回复，完成后再次发送 |

## 📝 总结

KoriChat 支持两大类指令：

1. **调试命令**：用于管理和调试机器人
   - `/help`、`/mem`、`/reset`、`/clear`、`/context`、`/gen_core_mem`、`/exit`

2. **内容生成命令**：生成角色相关的内容
   - `/diary`、`/state`、`/letter`、`/list`、`/pyq`、`/gift`、`/shopping`

这些指令可以帮助你：
- ✅ 了解机器人的内部状态
- ✅ 管理和调试记忆系统
- ✅ 生成有趣的角色内容
- ✅ 增加互动乐趣

**使用建议**：
- 调试命令建议在私聊中使用
- 内容生成命令适合娱乐和互动
- 谨慎使用删除类命令（`/reset`、`/clear`）
- 享受与 AI 角色的互动乐趣！
